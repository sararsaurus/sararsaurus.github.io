react

I wanted to tackle a project that would help me really understand React Hooks, so I decided to refactor some outdated Codecademy code that uses classes rather than functions and repetitive component lifecycle code like componentDidMount and componentDidUpdate . Hooks were introduced in 2018 as an addition to React 16.8 and offered an elegant way to add (or hook into) React features without writing a class. And useEffect in particular eliminates the need for some repetitive code that instructs the browser what to render.

I simplified the project to start. I'll eventually turn this into a random color generator, but for now I'm just going to create a button that changes its background color when clicked. I did something similar with the animal facts app , but in that case I just manipulated the DOM directly via document.getElementById . This time we're going to do it the right way by using state. The funny thing is that I've used all these concepts in more sophisticated ways before, particularly with SomeTrails , but it takes doing and unpacking several times to really internalize it.

learnings

THM's Advent of Cyber started off strong this week with an intro to hardware hacking! We used a logic analyzer to examine the signals traveling between devices, and we learned about protocols for transmitting data: USART (serial communication), SPI (which separates the clock from the data line and so has an extra wire compared to USART, but is faster and more reliable), and I2C (which was created to mitigate the drawbacks of both USART and SPI - over a thousand devices can be connected to the same two lines). Without protocols, we'd need an unfathomable number of wires to transmit the binary representation of each ASCII character ).

Then we had fun with IoT devices, which of course are uniquely vulnerable because they typically must be lightweight, without human-enabled authentication (like a manually typed password), and designed to all be interconnected. Security is often at best a secondary consideration. IoT devices typically rely on middleware to support their messaging protocol, such as MQTT, CoAP, AMQP, or DDS. (And then I had a little moment of panic where I realized I didn't fully understand what a port is...it's just a virtual point where network connections start and end, software-based and managed by a computer's OS.) And we interpreted some device source code to figure out its expected behavior so we could send a message and change its behavior - which required puzzling out some JSON hashes!

learnings

We did a lot during this week's THM Advent of Cyber challenges. We dove into malware's classic patterns, like establishing network connections, using registry keys, and downloading/creating files. We tackled the layers of defense necessary to secure file uploads and user inputs on a web app by learning about file extension validation, file size validation, file renaming, and malware scanning and by practicing (in PHP) wrapping GET requests in a function that converts the input to an integer and using a prepared statement for our search function to separate the syntax of our SQL sentence from our WHERE parameters for file content validation.

We also learned about the OWASP Top 10 list - specifically IDOR vulnerabilities (where a user can manipulate poor access control to bypass authorization). And we used industry tools like Wireshark to conduct traffic and packet analysis, which is the process of extracting, assessing, and identifying network patterns such as connections, shares, commands and other network activities, like logins, and system failures, from prerecorded traffic files). And we learned a few Sigma rules to analyze logs and detect threats.

And, of course, regexes returned! Regexes, combined with HTML's built-in features, give us pretty granular control over user input. For example, input type="text" id="uname" name="uname" pattern="[a-zA-Z0-9]+" will filter for a case-insensitive alphanumeric string.

I've been working on a terminal to-do app, and to create my incomplete tasks array I learned about Ruby's .select method. Turns out using .select on an array always creates a new array (therefore rendering unnecessary my first attempt with a blank array and a conditional statement). Select is great when you want to filter a list and return an array with the results: @tasks.select { |task| !task.completed } .

Day 8 of THM's Advent of Cyber tackled smart contracts... which are only as smart (and as secure) as their programmed logic. A common vulnerability is re-entrancy, which occurs when a malicious contract exploits flawed logic by using a fallback function to continue depleting a contract's total balance. This was my first encounter with Solidity, an OOP language created specifically for smart contracts on blockchain platforms.

I was trying to validate ISBNs and needed to make sure that the string contained only digits until the last character. And lo, my first regex (regexp?): /\D/.match(isbn_string.chop) .
